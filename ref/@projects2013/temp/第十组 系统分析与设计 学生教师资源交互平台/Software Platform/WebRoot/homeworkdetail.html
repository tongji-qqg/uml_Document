1. Introduction

You are required to form a group project team with 3 to 5 members. Each team should develop a multiple tier enterprise application which will involve development of web pages, servlets and connection to a database. Third party frameworks and libraries are highly encouraged to use in your project. One example of the projects could be an online retail store which sells a variety of goods such as books, toys and CDs. The team project will include a final project report as well as a presentation in the end of this semester. The project outcome be evaluated on the report, presentation and the quality of the software developed. The goal of this project is to demonstrate your understanding of the technical components of the Java enterprise application.

4. Progress Reports  (10%)

5. Project Proposal and Presentation (10%)

In the fifth week, each group should present your project proposal to the instructor and all of the class members. The follwing issues should be covered in your presentation: 

Ø Clear description of your idea 

Ø Target and scope of your proposed project 

Ø Possible technical solution and work plan

Ø Reference architecture (optional)

6. Middle Term Assesment (25%)

6.1 Project progress report and Presentation (10%)

Ø The project progress

Ø What did you completed in the first iteration

Ø Important issues during the development

Ø Next period planning

6.2 First Prototype Demonstration (15%)

Ø Demonstrate the basic functionalities implemented in the first prototype

Ø Explain the key component implementation

7. Final Implementation, Presentation, & Code Review  (55%)

The final project will consist of complete code, documentation, presentation and demonstration. 

7.1 Presentation (15%)

Ø Review your project

Ø Explain the whole architecture of your application

Ø Explain the key techniques used in your application

Ø Self Assessment

7.2 Demonstration (20%)

The demonstration should show the functionality of your website service. You should show the components created (if not enough time allows- at least major servlets and javabeans) as well as demonstrate how a new user would approach the site. Detail why everything was done and make it clear where something is "mocked" (using a static page) and not done using dynamic pages. 

7.3 Application and Documentation (20%)

The objectives of the application you are creating should include: 

Ø Ability to design and implement e-business and e-commerce solutions using the Java programming language 

Ø Ability to use knowledge and understanding to synthesize J2EE design solution website 

Ø Enough content in the website to demonstrate the functionality of the website 

Technical Features: 

Ø An Access database back end, which holds the content. You must write and read from the database. You are encouraged to use open source O/R mapping framework

Ø The use of servlets or Session EJB to communicate and possibly control the application 

Ø The use of JSPs for dynamic content presentation. These should also include, as appropriate, page includes for your navigation menu and other repeated content on the website 

Code Assessment 

The correctness of the code as well as your adherence to proper code writing style as well as design will be considered while evaluating your projects/code.
